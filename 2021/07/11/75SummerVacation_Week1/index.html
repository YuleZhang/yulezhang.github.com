本周是我们暑假特训的第一周，重点阅读了《Number-State Preserving Tensor Networks as Classifiers for Supervised Learning 》和《Generative tensor network classification model for supervised machine learning 》这两篇论文，并对这两篇论文进行了细致的理解和拓展，下面记录一些主要的内容和思路。

Number-State Preserving Tensor Networks as Classifiers for Supervised Learning [Paper]

论文回顾

翻译后是“数字态保持的张量网络用于监督学习分类”，因此本文的核心体现在数字态保持，下面将分别从数字态和数字态保持这两个方面来简单梳理下全文脉络。

  • 数字态∣ZL⟩\left|\mathcal{Z}^{\mathcal{L}}\right\rangle∣∣∣​ZL⟩意味着将经典数据用一系列基态的直积{∣0⟩,∣1⟩,…,∣d−1⟩}\{|0\rangle,|1\rangle, \ldots,|d-1\rangle\}{∣0⟩,∣1⟩,…,∣d−1⟩}来表示，即

∣ZL⟩=∣z0⟩∣z1⟩∣z2⟩…\left|\mathcal{Z}^{\mathcal{L}}\right\rangle=\left|z^{0}\right\rangle\left|z^{1}\right\rangle\left|z^{2}\right\rangle \ldots ∣∣∣​ZL⟩=∣∣∣​z0⟩∣∣∣​z1⟩∣∣∣​z2⟩…

  • 数字态保持指的是通过一些transformation，可以将一个数字态转换到另一个数字态上。这样的转换主要基于方向张量(oriented tensor)，w,u,vw, u, vw,u,v是不同的方向张量，将其reshape成为矩阵后表示如图1所示：


    图1 数字态保持张量

    这些方向张量必须满足以下两个基本条件才能保证输入输出都为数字态， 即方向张量reshape后的矩阵必须满足

    • 每行只能有一个非零元素(不一定是1)
    • 每一列可以有很多个非零元素

    本文在上述数字态张量网络定义的基础上设计了MERA结构T\mathcal{T}T，该结构由**方向张量w,u,vw, u, vw,u,v**组成，表示如2所示，将输入态⟨Zin∣\left\langle\mathcal{Z}^{\mathrm{in}}\right|⟨Zin∣∣∣​经⟨Zin∣T↦⟨Zout∣\left\langle\mathcal{Z}^{\mathrm{in}}\right| \mathcal{T} \mapsto\left\langle\mathcal{Z}^{\mathrm{out}}\right|⟨Zin∣∣∣​T↦⟨Zout∣转换为输出态⟨Zout∣\left\langle\mathcal{Z}^{\mathrm{out}}\right|⟨Zout∣。⟨Zout∣\left\langle\mathcal{Z}^{\mathrm{out}}\right|⟨Zout∣态会继续与标签yky_kyk​收缩。


    图2 数字态保持MERA结构

当然仅定义数字态张量网络还不够，论文中还提了这样网络的权重更新方式"跳变"。只有"跳"着更新权重张量(w,uw, uw,u)才能使其始终保持数字态张量的特性。这一点与传统的连续的权重张量相区别。也正因此，传统的连续张量采用的反向传播、梯度更新方式在这里并不适用。定义的跳变更新方法也很难应用到经典的权重张量更新中去。跳变更新的关键在于计算权重张量(以uuu为例)的环境张量Γu\Gamma_{u}Γu​，下面将进行介绍。

测试集预测准确的数目是通过将**MERA结构T\mathcal{T}T、输入张量$$\left\langle\mathcal{Z}^{\mathrm{in}}\right|$$和输出张量$$\left\langle\mathcal{Z}^{\mathrm{out}}\right|$$**三者的收缩得到的，如下所示。

Ncorrect =∑k⟨Zkin∣T∣yk⟩=tr⁡(u⋅Γu†)N_{\text {correct }}=\sum_{k}\left\langle\mathcal{Z}_{k}^{\mathrm{in}}|\mathcal{T}| y_{k}\right\rangle=\operatorname{tr}\left(u \cdot \Gamma_{u}^{\dagger}\right) Ncorrect ​=k∑​⟨Zkin​∣T∣yk​⟩=tr(u⋅Γu†​)

其中Γu†\Gamma_{u}^{\dagger}Γu†​是权重张量uuu的环境张量，与环境张量同阶，它是收缩除了张量uuu的MERA中其他所有张量得到的，因此NcorrectN_{correct}Ncorrect​可以通过权重矩阵和它环境张量的收缩计算得到。对于方向张量uuu的更新，文中提到了两种方式，一种是选每一行最大的。另一种是为了避免过拟合而设计的，将每一行的数据表示概率的方式，用了下面的公式进行映射

pijtrans=exp⁡(Ωij/α)∑jexp⁡(Ωij/α)p_{i j}^{\mathrm{trans}}=\frac{\exp \left(\Omega_{i j} / \alpha\right)}{\sum_{j} \exp \left(\Omega_{i j} / \alpha\right)} pijtrans​=∑j​exp(Ωij​/α)exp(Ωij​/α)​

这种方式也是一种不错的行归一化方式，也许可以用到其他网络收缩时的归一化模块中。

Generative tensor network classification model for supervised machine learning [Paper]

这篇论文之前已经看过了，并且处理了很久相关的代码，但对论文的细节已经比较模糊了。在阅读这篇论文的同时，我们还仔细阅读了一下TSGO模型《Tangent-Space Gradient Optimization of Tensor Network for Machine Learning》。在本次阅读中，TSGO花了不少的功夫，主要是有一部分公式的处理逻辑不是很清晰。就简单再梳理一下公示的逻辑，TSGO是一种优化模型，但它的应用范围不是无限制的。

要想采用TSGO优化模型，必须满足条件B或者条件C，如图4所示。条件B可以通过条件C证出，当然也可以通过其他一些条件证出，只不过论文中没有提到罢了。也就是条件B是模型能否采用TSGO优化的关键条件。


图4 TSGO优化模型核心

论文中采用正交的MPS举例，采用TSGO原理对其优化，是因为正交MPS的核心张量TTT与它的梯度内积为0，即⟨∂f∂T[n~],T[n~]⟩=0\left\langle\frac{\partial f}{\partial T^{[\tilde{n}]}}, T^{[\tilde{n}]}\right\rangle=0⟨∂T[n~]∂f​,T[n~]⟩=0。同时为了弄清楚论文中导数出现的2(如图5)，我们找到了另一篇高质量文章《Unsupervised Generative Modeling Using Matrix Product States 》[Paper] (2018)，包含了更详细的说明。

同时在阅读GTNC的过程中，我尝试手动写一下Lazy Learning的代码，思路很简单。但是在写的过程中发现了很重要的一个问题，如图5所示，即使我们将懒惰学习优化成为O(NLd)O(NLd)O(NLd)的多项式复杂度，但仍存在指数消失的问题。这个问题是由红框标注出来的累乘导致的，这使得像素内积的概率指数小，算出的概率没有意义(基本都为0)。


图5 懒惰学习无法求解问题

对于该问题，我们咨询了首师大冉老师，并收到了他及时的指点。冉老师的回复原文如下

你的计算没有问题，那么化简的计算复杂度是O(NLd)。可以通过加上一个小量保证稳定性，可以参考我们最新的文字arxiv.2107.00195中公式（3），里面的epsilon项。

原文名为《Non-parametric Active Learning and Rate Reduction in Many-body Hilbert Space with Rescaled Logarithmic Fidelity》[Paper]。本来我以为可能就是一个参数ϵ\epsilonϵ引入常量，还是没有实质性解决累乘的问题。直到我看了一下这篇文章，发现里面完全是详细的说明了一下在表示映射到Hilbert空间的样本时出现的指数爆炸问题，并引入log10log_{10}log10​函数对其进行化简，巧妙的将累乘化为求和的形式，原文关键内容如图6所示


图6 样本在Hilbert空间的指数化简

同时这里也提到了一种比较有意思的、新颖的距离计算方式，如图6中公式（2）所示，直接通过原始像素之间的距离来计算样本保真度。这个思路可以用到张量网络的距离计算中去。这也将是下一步进行学习和实验的目标。
